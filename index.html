Getting Started
Base URL and authentication for all Acrobat Sign REST v6 API calls. Set your Bearer Token in the sidebar to enable live testing.
Base URL
https://{baseUri}/api/rest/v6/
Always retrieve your base URI dynamically via GET /baseUris. It differs by region (na1, na2, eu1, ap1).
Authentication Header
Authorization: Bearer <access_token>
Standard Request Headers
Content-Type: application/json
Authorization: Bearer <token>
x-api-user: email:user@example.com  // optional: act on behalf of user
Core Workflow
// 4-step signing workflow
1. POST /transientDocuments       // get transientDocumentId
2. POST /agreements               // get agreementId
3. GET  /agreements/{id}          // poll status or use webhook
4. GET  /agreements/{id}/combinedDocument  // download signed PDF
💡 Set your Bearer Token in the sidebar, then visit any endpoint's Test tab to fire live requests. All responses also appear in the Response Log.

Rate Limits & Throttling
Acrobat Sign enforces four throttling mechanisms. All return HTTP 429 when exceeded.
⚡ HTTP 429 — REST API Rate Limit ▼
429 Responses
Schema
Polling (GET)
System Load
{
  "code": "THROTTLING_TOO_MANY_REQUESTS",
  "message": "...try again in <N> seconds",
  "retryAfter": 60
}
Retry-After: 60   // also in response header
{
  "code": enum        // REQUIRED — THROTTLING_TOO_MANY_REQUESTS | THROTTLING_HIGH_SYSTEM_LOAD
  "message": string     // REQUIRED
  "retryAfter": integer  // REQUIRED — seconds to wait
}
// Headers:
"Retry-After": integer
"X-Throttling-Reason": string  // optional — "high-system-load"
// GLOBAL/ENTERPRISE tier: 3 identical calls per 1-minute interval
// All other tiers: 1 call per 10-minute interval

// Best practice — conditional GET with ETag:
GET /api/rest/v6/agreements/{id}
If-None-Match: "D0CA32AB5F61126FA..."
// 304 Not Modified if unchanged | 200 + new ETag if changed
⚠️ Use webhooks for real-time updates. Excessive polling may result in temporary API suspension.
{
  "code": "THROTTLING_HIGH_SYSTEM_LOAD",
  "message": "...try again in <N> seconds.",
  "retryAfter": 30
}
X-Throttling-Reason: high-system-load
A retry penalty applies for accounts created after May 20, 2025: retrying before the Retry-After window resets the timer. Always use the server-provided delay.
